What is FusionAuth Reactor?

Built from a fire-forged, vibranium-authenticatium alloy, FusionAuth Reactor is a suite of new features that extends FusionAuth's core functionality.

When Reactor is activated, FusionAuth can connect to powerful cloud-based components that address specific identity challenges. Some focus on security, others on saving time in complex integrations. Enable one feature or the whole suite depending on your specific requirements.

FusionAuth Reactor is Native Code

FusionAuth Reactor is native code, not a set of plug-ins or modules that get strapped onto the side of the core FusionAuth platform. We’ve seen this in other identity products, and know that it can hurt your speed, efficiency and security.

Reactor components are architected directly into the FusionAuth platform so you don't make sacrifices for additional features. Our mission is to provide a fast, efficient, and secure identity solution that can work anywhere, and everything in Reactor keeps this in focus for every component.

FusionAuth Reactor Is Encrypted and Secure

FusionAuth Reactor's cloud-based components are hosted securely behind our vibranium-authenticatium firewall for bank-level security. All data is end-to-end encrypted for complete protection.

FusionAuth Reactor Components

Each FusionAuth Reactor component addresses specific challenges within identity and access management. Here's what's available now.

Breached Password Detection

The first feature available as part of Reactor is Breached Password Detection. We collected hundreds of millions of compromised usernames and passwords from numerous breached databases. FusionAuth can now check passwords to ensure that they don’t exist in any of these databases.

Breached Password Detection can be enabled during these events:

  • Account creation
  • Password reset or change
  • During login

This flexibility allows for retroactive password clean-up for existing users as well as restricting the use of breached passwords for new users. When Reactor determines that a password has been breached, FusionAuth can be configured to take one of these actions:

  • Force a password change
  • Notify the user by email
  • Mark the user as having a breached password for later reporting and action

Prevent Brute Force Attacks

By activating this feature and combining it with our already powerful password constraints, you can ensure your users are secure and using the latest recommended password rules. Plus, as new breached databases become available on the dark web, we’ll be adding them to Reactor and your FusionAuth will automatically be updated. This combination will help keep your users secure and prevent the most common brute force attack strategies.
