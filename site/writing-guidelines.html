Writing guidelines

Thanks for writing for us. Here are some guidelines to help make the process go smoothly.
  • The typical process is:
    1. Contact is made.
    2. We discuss a few topics based on our needs and your interests.
    3. We agree on a topic.
    4. You send over an outline.
    5. I give feedback and we discuss the outline, and then I approve it via email.
    6. You write the article.
    7. You deliver it.
    8. I give feedback or ask questions after review. Typically we do 1-2 revisions.
    9. I approve it.
    10. I get payment scheduled.
    11. We publish the article.
  • Please provide your paypal email address, as that's how we'll pay you.
  • We prefer to have the article delivered in markdown with images in a zip file. If that doesn't work, then google docs is fine, but we still need the images separately.
  • Use the authorization code grant if you are working with OAuth. Please use the "state" parameter to defend against CSRF attacks.
  • Make sure you store all tokens securely, as outlined here.
  • If you make an example application, we'd love to have that transferred over to our github repository, where we'll make it available under the Apache license. If you'd like to add yourself as a contributor in the project readme, please feel free to do so.
  • When you provide images, please make them available as a zip file. We post-process images included in our posts.
  • If you take screenshots of the FusionAuth interface, please don't spend a lot of time on making them look good. We re-do them in house with a given configuration. Just give us enough information that we can recreate it.
  • If you make a video of any part of the process, please give us the source file so we can upload it to our youtube channel.
  • We'll notify you when the article is published, and would appreciate any social sharing you can do.
